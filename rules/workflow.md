# Workflow Rules

## Standard lifecycle

The standard flow is:

1. requirement intake
2. exploration and impact analysis
3. brainstorming and scope confirmation
4. design
5. plan
6. TDD execution
7. review and verification
8. archival of artifacts

## Artifact expectations

- `harness/explorations/` stores research and impact analysis
- `harness/work/` stores active work slices
- `harness/changes/` stores formal change proposals
- `harness/specs/` stores stable specifications

## Phase-1 emphasis

The first phase prioritizes workflow standardization over full autonomy.
