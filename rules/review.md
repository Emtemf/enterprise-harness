# Review Rules

## Hard-gate reviewers

Phase 1 uses five hard gates:

1. requirement correctness
2. design completeness and architecture fit
3. plan executability and clarity
4. API contract consistency
5. verification evidence

## Soft-gate reviewers

Advisory reviewers may recommend:

- architecture cleanup
- stronger test design
- rollout and release considerations

## Review principle

The minimal hard-gate set blocks incorrect scope, poor plans, broken contracts, and unverified delivery without turning the MVP into a heavy process.
